## 1. Текущий статус Бэкенда (Backend Status)
- На данный момент бэкенд (Python) работает **синхронно**. Эндпоинта `/status` пока нет.
- **ВНИМАНИЕ:** Мы приняли архитектурное решение **ОТКАЗАТЬСЯ от WebSockets**. Вместо этого бэкенд будет использовать **Server-Sent Events (SSE)** и стандартный HTTP Streaming. Это индустриальный стандарт для передачи ответов от LLM.

## 2. Стратегическое решение: Переход на Next.js (React)
Мы переписываем фронтенд на **Next.js (App Router)** + **shadcn/ui** + Tailwind.
В идеале стоит использовать **Vercel AI SDK** (`useChat` или `useCompletion`), так как он из коробки идеально работает с SSE и стримингом текста.

## 3. Архитектура: SSE & Stateless UI
- Фронтенд должен управлять состоянием через React State.
- Вместо сложного поднятия WebSocket-соединения, UI будет делать обычный HTTP-запрос и читать `ReadableStream` (SSE), плавно добавляя текст и обновляя статусы (Pending -> Generating -> Validating -> Done) в timeline.

## 4. Управление контекстом (КРИТИЧНО)
- Фронтенд отправляет строго два раздельных поля в JSON:
  - `prompt`: текстовый запрос пользователя.
  - `context`: (опционально) текущий Lua-код.
- Ни в коем случае не склеивайте текст и код в одну строку на клиенте!
